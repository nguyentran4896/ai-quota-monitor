# Support

Use GitHub Discussions for setup questions and the issue templates for confirmed
bugs or feature proposals. Before filing a bug, run `pnpm check` on a source
checkout or note the installed QuotaDeck version, operating system, desktop
environment, provider CLI version, and whether the profile is managed.

Use the private reporting link for security or conduct concerns; start conduct
report titles with `[CONDUCT]` so maintainers route them under the code of conduct.

Never attach provider credential files, `auth.json`, `.credentials.json`, complete
session logs, or Claude transcripts. QuotaDeck contributors cannot restore or
reset provider accounts; use the provider's official support channel for account
and billing problems.
