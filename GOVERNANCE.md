# Governance

QuotaDeck currently uses a maintainer-led model. Maintainers review changes,
publish releases, manage security reports, and protect the project's local-first
security boundary. Routine decisions happen in issues and pull requests. Major
changes to credential handling, provider integrations, telemetry, licensing, or
the desktop architecture require a written decision under `docs/` before code is
merged.

As the contributor group grows, maintainership may be granted based on sustained,
constructive contributions and sound security judgment. No contributor may merge
their own security-sensitive change without another maintainer's review.
